Form Settings

The settings level defines a form and attributes that determine the form's behavior.

Attribute type description
Title Text Title for your form
Title Font Size Text Title font size in px
Description Textarea Description for your form
Description Font Size Text Discription font size in px
Action Text The action attribute defines the action to be performed when the form is submitted. Usually, the form data is sent to a file on the server when the user clicks on the submit button.
Method Text The method attribute specifies the HTTP method to be used when submitting the form data. The form-data can be sent as URL variables (with method="get") or as HTTP post transaction (with method="post").
Encoding Type Text The enctype attribute specifies how the form-data should be encoded when submitting it to the server.
Note The enctype attribute can be used only if method="post".
ID Text Add a custom DIV ID
Form ID Text Specifies the form element the input element belongs to. The value of this attribute must be the id attribute of a form element in the same document.
Database Action Picklist How would you like to edit your database. You can create, update or delete.
Database Object Text What database object are you modifying. Example: user__c
Record ID Text What record do you want to target when updating or deleting a record.
Redirect URL Text After successfully completeing a form, where would you like to redirect the user
Flash Message Text Message to display after successfully compeleting a form
Custom Button Label Text Custom button will appear at the top of your form with this text
Custom Button Link Text The URL for your custom button
Hide Submit Button Boolean Removed the submit button from the form
Submit Label text Change the default 'Submit' text
Enable Cancel Button Boolean Display the cancel button
Cancel Label text Change the default 'Cancel' text

Form Fields

Granite has a wide variety of form fields to accomidate all your needs. The following documentation will breakdown the list of fields.

Global Attributes

All form fields have a global set of attributes for basic functionality. In additional to the global attributes, some fields have custom functionality that require custom attributes.

Attribute Type Description
Field Title Text Used for the field label
Field Name Text Used for the name attribute
Field Type Picklist Specifies the field type
Field Value Text - Ruby variable Assign the value from your connected database ({record.}) or hardcode a value
Field Required Boolean Specifies that an input field must be filled out before submitting the form.
Classes Text Add custom classes for targeting the field
ID Text Add a custom ID to target
Disabled Boolean Specifies that the field should be disabled
Display Inline Boolean Displays the next form field inline with the current field
Dependency Text Target the parent field using that fields name attribute
Dependency Values Text The parent field value(s) that will display the field.
Display Inline with Next Field - NEW boolean The Next form field will display on the same row. You can repeat this as many times as youd like.
Custom Regex Pattern - NEW text Override the default HTML validation check with your own custom Regular Expression
Invalid Message - NEW text Override the default HTML validation error with your own text

Dependency

The name attribute is used to set the parent dependecy field. The following field types can be used as a parent dependecy field: checkbox, text and picklist.

The dependent field can accept a boolean values (true or false) single text values or multiple values separated by a comma.

Depedency Value(s): Texas, Montana

Dependency Checkbox Demo

Depedency Value: True

Dependency Text Demo (Mutiple Values)

Depdendency Field: fruit (The fields name attribute)

Depedency Value(s): Orange, Appple, Banana

Color Picker

The color picker interface has two elements that lets a user specify a color, either by using a visual color picker interface or by entering the color into a text field in #rrggbb hexadecimal format.

Attributes Specific to Color Picker

Attribute Type Default Accepted Values
Unit text px Any value
Min Value number 0 Any number
Max Value number 100 Any number

Checkbox

The checkbox is shown as a square box that is ticked (checked) when activated.

Date

The date field currently uses jQuery UI as a dependecy due to the lack to browser support. With that being said, native functionaloty on mobile devices is overridden.

File

let the user choose one or more files from their device storage.

Attributes Specific to File

Attribute Type Default Accepted Values Description
accept text all file types image/*, audio/*, video/*, .pdf, .png, .doc, docx, .xml Attribute takes as its value a string containing one or more of these unique file type specifiers, separated by commas.

Email

The input value is automatically validated to ensure it is a properly formatted e-mail address.

Attributes Specific to Email

Attribute Type Description
Multiple Boolean Allows multiple e-mail addresses when true

Hidden Field

A hidden field lets developers include data that cannot be seen or modified by users when a form is submitted. This often stores what database record that needs to be updated when the form is submitted.

Number

They number field include built-in validation to reject non-numerical entries. The User may opt to provide min and max values.

Attributes Specific to Number

Attribute Type Description
Show Counter Boolean Hide the counter for increasing or decreasing the number
Stepper (Increase or Decrease Number by) Number Specifies the legal number intervals
Max Number Number Specifies the maximum value allowed
Min Number Number Specifies the minimum value allowed

Password

provide a way for the user to securely enter a password. The element is presented as a one-line plain text editor control in which the text is obscured so that it cannot be read.

Attribute Type Description
Show Password Boolean Hide or show the characters in the password field

Phone

used to let the user enter and edit a telephone number. Unlike email and url , the input value is not automatically validated to a particular format before the form can be submitted, because formats for telephone numbers vary so much around the world.

We recommend using the Custom Regex Pattern attribute if youd like a formatted number.
